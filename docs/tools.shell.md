# Shell

Although separated by differences, the core of shell environments remain intact.
We run programs and, occasionaly string a bunch together.

I use whatever is at hand, and have no preference.
But I use these 2 things the most.



## 1. Running Programs

Any program devised on earth needs to be runnable.
This is perhaps the most pleasant way to do so.

```sh
$ program
```


## 2. Redirecting Input & Output

Input & output of a program should be invisible to it.
It comes and goes, it should not mind where.
I often put output of a program into a file, or supply a file as input to another.

Common operators such as `<`, `>` and `|` are elegant in their design, and I put them to use when needed.
I subconciously check these two rules to make a decision.

- If it makes things less clear, I avoid them.
- If it makes things less simple, I avoid them.
- I think before I type.


## Conclusion

Shell is great for one thing: running programs interactively.
Nothing else.
